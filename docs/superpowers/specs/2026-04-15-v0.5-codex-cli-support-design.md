# v0.5.0 — Codex CLI 플랫폼 지원

## 목표

so2x-harness에 OpenAI Codex CLI를 두 번째 플랫폼으로 추가한다.
한 프로젝트에 Claude Code + Codex CLI를 동시 설치할 수 있도록 한다.

## 배경

현재 harness는 Claude Code만 지원한다 (`--platform claude`).
Autopus-ADK 등 멀티 플랫폼 harness가 이미 존재하며, 사용자가 여러 AI 코딩 도구를 병행하는 사례가 늘고 있다.

## 플랫폼 매핑

| 개념 | Claude Code | Codex CLI |
|------|------------|-----------|
| 지시 파일 | `CLAUDE.md` | `AGENTS.md` (marker 기반) |
| 스킬 | `.claude/skills/` | `.agents/skills/` |
| 규칙 | `.claude/rules/` | AGENTS.md 내 섹션으로 병합 |
| 훅 | `.claude/hooks/` | 없음 (미지원) |
| 에이전트 | `.claude/agents/` | 없음 (미지원) |
| 설정 | `.claude/settings.json` | `~/.codex/config.yaml/json` (harness 미관여) |

## 설계

### 1. platform_map.py 확장

`PROJECT_PATHS` 딕셔너리에 `"codex"` 항목을 추가한다.
모든 경로 접근은 `dict.get()`을 사용하며, 없는 키는 `None`을 반환한다.
호출 측(apply.py, update.py, doctor.py)은 `None` 체크 후 해당 단계를 건너뛴다.

```python
PROJECT_PATHS = {
    "claude": {
        "rules_dir": Path(".claude/rules/so2x-harness"),
        "skills_dir": Path(".claude/skills"),
        "agents_dir": Path(".claude/agents/so2x-harness"),
        "hooks_dir": Path(".claude/hooks"),
        "shared_docs_dir": Path(".ai-harness/docs"),
        "shared_snippets_dir": Path(".ai-harness/snippets"),
        "agents_path": Path("AGENTS.md"),
        "claude_md_path": Path("CLAUDE.md"),
        "config_path": Path(".ai-harness/config.json"),
    },
    "codex": {
        "skills_dir": Path(".agents/skills"),
        "shared_docs_dir": Path(".ai-harness/docs"),
        "shared_snippets_dir": Path(".ai-harness/snippets"),
        "agents_path": Path("AGENTS.md"),  # codex 템플릿 사용 (rules 병합)
        "config_path": Path(".ai-harness/config.json"),
    },
}

# 플랫폼별 지원 기능 플래그
PLATFORM_CAPABILITIES = {
    "claude": {"rules": True, "skills": True, "agents": True, "hooks": True},
    "codex": {"rules": False, "skills": True, "agents": False, "hooks": False},
}
```

apply.py, update.py, doctor.py에서 경로에 직접 인덱싱(`paths["rules_dir"]`)하는 모든 코드를
`paths.get("rules_dir")`로 변경하고, `None`이면 해당 단계를 건너뛴다.
또는 `PLATFORM_CAPABILITIES`를 확인하여 기능 지원 여부로 분기한다.

### 2. 템플릿 구조

```
templates/
├── claude/              # 기존 그대로 유지
│   ├── CLAUDE.md
│   ├── agents/
│   ├── rules/
│   ├── skills/
│   └── hooks/
├── codex/               # 신규
│   ├── AGENTS.md        # shared/AGENTS.md + rules 내용 병합 (managed block 포함)
│   └── skills/          # claude/skills 와 동일 스킬 파일
└── shared/              # 기존 그대로
    ├── AGENTS.md
    ├── docs/
    └── snippets/
```

#### AGENTS.md 병합 규칙

`templates/codex/AGENTS.md`는 다음 순서로 구성한다:

1. `templates/shared/AGENTS.md` 본문
2. 빈 줄 + `## Rules` 섹션 헤더
3. `templates/claude/rules/*.md` 파일 내용을 각각 `### <rule-name>` 하위 섹션으로 삽입

이 파일은 사전 생성(커밋 시점)하여 템플릿에 포함한다.
apply 시점에 동적 병합하지 않는다 — 템플릿이 단일 소스다.

#### AGENTS.md managed block

`templates/codex/AGENTS.md`와 `templates/shared/AGENTS.md` 모두 `SO2X-HARNESS` marker block을 포함한다:

```markdown
<!-- SO2X-HARNESS:BEGIN -->
... managed content ...
<!-- SO2X-HARNESS:END -->
```

apply/update 시 marker 유틸(`upsert_marker_block`)을 사용하여 관리 구간만 교체한다.
사용자가 marker 외부에 작성한 로컬 내용은 보존된다.

#### 다중 플랫폼 설치 시 AGENTS.md 처리

AGENTS.md는 두 플랫폼이 공유한다. 다중 플랫폼 설치 시 처리 규칙:

- codex가 설치 대상에 포함되면 AGENTS.md에 **항상 codex 템플릿**(rules 병합본)을 사용한다.
- codex 템플릿은 shared/AGENTS.md 내용을 포함하므로 claude에서도 문제없다.
- claude 단독 설치 시는 기존처럼 shared/AGENTS.md를 사용한다.
- 기존 AGENTS.md에 사용자 로컬 내용이 있으면(marker 외부) 보존한다.

#### Skills

`templates/codex/skills/` 디렉터리에 `templates/claude/skills/` 와 동일한 스킬 파일을 둔다.
내용이 플랫폼에 무관하므로 파일을 복제한다.
claude 스킬이 변경되면 codex 스킬도 수동으로 동기화해야 한다.
parity 검증은 테스트로 자동화한다 (`test_codex_skill_parity`).

### 3. apply.py 변경

#### 인자 변경

```python
parser.add_argument(
    "--platform",
    nargs="+",
    default=["claude"],
    choices=["claude", "codex"],
)
```

- 기본값은 `["claude"]`로 deterministic하다.
- 자동 감지는 하지 않는다 (install 스크립트에서 담당).
- 중복 제거: `list(dict.fromkeys(args.platform))`으로 동일 플랫폼 중복 입력을 방지한다.

#### 다중 플랫폼 설치 흐름

```python
platforms = list(dict.fromkeys(args.platform))  # 중복 제거, 순서 유지
for platform in platforms:
    apply_platform(project, platform, preset, root)
```

`apply_claude()`를 `apply_platform(project, "claude", ...)` 로 리네이밍하고,
플랫폼별 분기는 `platform_map.py`의 경로 딕셔너리와 `PLATFORM_CAPABILITIES`로 처리한다.

공통 로직 (디렉터리 생성, 파일 복사, marker 삽입, manifest 업데이트)은 헬퍼 함수로 추출한다.
플랫폼별로 지원하지 않는 기능은 `PLATFORM_CAPABILITIES` 확인 후 건너뛴다.

#### manifest platforms 배열

```json
{
    "platforms": ["claude", "codex"],
    "files": { ... }
}
```

- 첫 설치 시 `platforms`에 선택된 플랫폼을 모두 기록한다.
- 재실행 시 기존 `platforms`에 **추가만** 한다 (제거하지 않는다, add-only 정책).
- 중복은 허용하지 않는다 (`list(set(...))`로 정리).

### 4. update.py 변경

- manifest의 `platforms` 배열을 읽어서 각 플랫폼을 순회하며 업데이트한다.
- 각 플랫폼의 `PLATFORM_CAPABILITIES`를 확인하여 지원하지 않는 기능은 건너뛴다.
  - codex: rules, agents, hooks 디렉터리 업데이트 건너뛰기

### 5. doctor.py 변경

- manifest의 `platforms` 배열을 읽어서 각 플랫폼별 진단 항목을 추가한다.
- codex 진단 항목:
  - `.agents/skills/` 존재 및 파일 수
  - `AGENTS.md` 존재 및 marker 확인
  - `.ai-harness/config.json` 존재
- `PLATFORM_CAPABILITIES`를 참고하여 미지원 기능에 대한 진단은 건너뛴다.

### 6. install.sh / install.ps1 변경

#### install.sh: 자동 감지 + 인터랙티브 선택

```sh
# 감지
detect_platforms() {
  found=""
  command -v claude >/dev/null 2>&1 && found="claude"
  command -v codex >/dev/null 2>&1 && found="$found codex"
  echo "$found"
}

# PLATFORM 환경변수가 없으면 인터랙티브
if [ -z "${PLATFORM:-}" ]; then
  DETECTED=$(detect_platforms)
  if [ -z "$DETECTED" ]; then
    DETECTED="claude"
  fi
  printf '감지된 플랫폼: %s\n' "$DETECTED"
  printf '설치할 플랫폼을 선택하세요:\n'
  printf '  1) claude\n'
  printf '  2) codex\n'
  printf '  3) 둘 다\n'
  printf '선택 [1-3]: '
  if [ -t 0 ]; then
    read -r choice
  else
    # non-TTY: 감지된 플랫폼 자동 선택, 없으면 claude
    choice=1
    echo "$choice"
  fi
  case "$choice" in
    1) PLATFORM="claude" ;;
    2) PLATFORM="codex" ;;
    3) PLATFORM="claude codex" ;;
    *) PLATFORM="claude" ;;
  esac
fi
```

- `PLATFORM` 환경변수가 설정되어 있으면 인터랙티브 프롬프트를 건너뛴다.
- non-TTY 환경(pipe)에서는 `read` 대신 기본값 사용.
- 감지된 플랫폼이 없으면 claude를 기본값으로 사용.

#### install.sh: apply.py 호출

```sh
"$PYTHON_BIN" "$ROOT_DIR/scripts/apply.py" \
  --project "$PROJECT_DIR_ABS" \
  --platform $PLATFORM \
  --preset "$PRESET"
```

POSIX shell에서 `$PLATFORM`(따옴표 없음)은 공백 분리되어 `nargs="+"`에 정상 전달된다.

#### install.ps1: 자동 감지 + 인터랙티브 선택

```powershell
# 감지
$detected = @()
if (Get-Command claude -ErrorAction SilentlyContinue) { $detected += "claude" }
if (Get-Command codex -ErrorAction SilentlyContinue) { $detected += "codex" }
if ($detected.Count -eq 0) { $detected = @("claude") }

if (-not $Platform -or $Platform -eq "claude") {
    # 인터랙티브 선택
    Write-Host "감지된 플랫폼: $($detected -join ', ')"
    Write-Host "설치할 플랫폼을 선택하세요:"
    Write-Host "  1) claude"
    Write-Host "  2) codex"
    Write-Host "  3) 둘 다"
    $choice = Read-Host "선택 [1-3]"
    switch ($choice) {
        "1" { $Platform = "claude" }
        "2" { $Platform = "codex" }
        "3" { $Platform = "claude,codex" }
        default { $Platform = "claude" }
    }
}
```

#### install.ps1: apply.py 호출

```powershell
# PowerShell은 공백 문자열이 단일 argv로 전달되므로 개별 인자로 분리
$platformArgs = $Platform -split ','
$applyArgs = @(
    "$ResolvedRoot/scripts/apply.py",
    "--project", $projectAbs,
    "--preset", $Preset
) + @("--platform") + $platformArgs

& $pythonCmd @applyArgs
```

PowerShell에서는 배열 splatting(`@applyArgs`)을 사용하여 각 플랫폼을 개별 인자로 전달한다.
