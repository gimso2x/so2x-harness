# Spec: v0.2 Spec Engine + Core Agents

## Goal

so2x-harness에 hoyeon 스타일의 spec.json 파생 체인과 6개 코어 에이전트, Python CLI를 추가한다.
spec.json을 단일 소스 오브 트루스로 사용하여, Goal에서 Tasks까지 6단계 파생 체인을 통해 검증된 구현 계획을 수립한다.

## Key Decisions

1. **JSON 포맷** — hoyeon 스타일 spec.json. 프로그래밍 검증 가능, 파생 체인 추적 가능
2. **6단계 파생 체인** — L0:Goal → L1:Context → L2:Decisions → L3:Requirements → L4:Tasks → L5:Review
3. **6개 코어 에이전트** — Interviewer, Code Explorer, Spec Writer, Planner, Reviewer, Verifier
4. **Python CLI (so2x-cli)** — pip install 가능, spec.json 검증/관리
5. **이중 게이트** — CLI 구조 검증 + 에이전트 품질 검증
6. **append-only** — Tasks는 추가만 가능, 기존 항목 수정 불가

## Requirements

### R1: spec.json 구조
- meta 섹션: id, goal, status, created_at, updated_at
- chain 섹션: l0_goal, l1_context, l2_decisions, l3_requirements, l4_tasks, l5_review
- gates 섹션: 각 단계 간 게이트 상태 (pass/pending/blocked)
- l3_requirements에 scenarios와 verify 명세 포함
- l4_tasks에 requirement_refs로 역추적 가능

### R2: 파생 체인
- 6단계: Goal → Context → Decisions → Requirements → Tasks → Review
- 각 단계 간 게이트로 진행 제어
- 하위 단계는 상위 단계를 참조 (방향성)
- append-only: 기존 Tasks 수정 불가, 추가만 가능

### R3: 게이트 시스템
- 각 게이트는 CLI 검증 + 에이전트 검증 통과해야 함
- CLI 검증: 구조, 필수 필드, 타입 체크
- 에이전트 검증: 논리, 완결성, 품질
- 실패 시 어떤 이유로 실패했는지 명시

### R4: 코어 에이전트 (6개)
- Interviewer: 질문만 함, 구현 안 함 (L0, L2)
- Code Explorer: 코드 분석, 패턴 파악 (L1)
- Spec Writer: spec.json 작성/수정 (L3)
- Planner: 태스크 분해, 순서 배치 (L4)
- Reviewer: 품질/일관성/위험 검토 (L5)
- Verifier: 시나리오별 독립 검증 (실행 후)

### R5: Python CLI (so2x-cli)
- spec init: 새 spec.json 생성
- spec check --gate: 특정 게이트 검증
- spec validate: 전체 구조 검증
- spec status: 현재 파생 상태 조회
- spec guide: 각 레이어 필드 구조 안내
- pip install -e . 로 설치 가능

### R6: 스킬/hook 통합
- /specify 스킬: Interviewer → Code Explorer → Spec Writer 파이프라인
- /execute 스킬: Planner → Worker → Verifier 파이프라인
- spec-gate hook: spec.json 변경 시 자동 게이트 검증
- 기존 spec-validate 스킬은 so2x-cli spec validate로 대체

### R7: 기존 시스템과의 호환성
- spec-lite markdown은 계속 지원 (마이그레이션 경로 제공)
- 기존 스킬/hook은 그대로 작동
- agent 템플릿은 apply.py로 설치 가능

## Risks / Unknowns

- Claude Code 에이전트 시스템이 파생 체인을 안정적으로 실행할 수 있는지
- spec.json이 너무 무거워지면 AI가 컨텍스트에서 처리하기 어려울 수 있음
- 에이전트 간 핸드오프가 실패할 경우 복구 메커니즘 필요
- so2x-cli가 사용자 환경에 pip로 설치되어야 함 (현재는 Python 스크립트만 있음)

## Implementation Steps

1. spec.json JSON Schema 작성
2. so2x-cli 기본 구조 (pyproject.toml, main.py)
3. so2x-cli spec init 구현
4. 게이트 검증 로직 (gate_checker.py)
5. so2x-cli spec check --gate 구현
6. so2x-cli spec validate 구현
7. so2x-cli spec status, guide 구현
8. 에이전트 템플릿 6개 작성
9. /specify 스킬 작성 (파생 파이프라인)
10. /execute 스킬 업데이트 (검증 파이프라인)
11. spec-gate hook 업데이트
12. spec-lite → spec.json 마이그레이션 가이드
13. 통합 테스트 작성
14. CI/CD 업데이트
15. 문서 업데이트
